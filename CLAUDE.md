# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running Locally

No build step required. Open `index.html` directly in a browser:

```
open index.html
```

Or serve it with any static file server (e.g. `python3 -m http.server`).

## Architecture

This is a single-file static site generated by Claude Design. All content lives in `index.html` (~15 MB).

**Bundler system** — `index.html` embeds three `<script>` tags that act as a virtual filesystem:
- `__bundler/manifest` — JSON map of UUID → base64-encoded gzipped asset data
- `__bundler/template` — the actual HTML/CSS/JS application template
- `__bundler/ext_resources` — references to any external CDN resources

On page load, inline JavaScript reads the manifest, decompresses each asset via the browser's `DecompressionStream` API, constructs `blob:` URLs, substitutes them into the template, and injects the result into the DOM.

**Content** — five scroll-driven storytelling scenes for "Paper Portal", a second-hand books marketplace:
1. Hero intro
2. Book Transfer (marketplace concept)
3. Read Together (community)
4. Sustainability (environmental angle)
5. New Chapter (conclusion)

Design: dark background (`#141009`), gold accent (`#D4A85A`), parallax scroll effects.

## Editing the Site

`index.html` is **auto-generated by Claude Design** — do not hand-edit it. The file is ~15 MB with all assets embedded as base64 inside the manifest, making direct edits impractical. To change content or design, update the prompt in Claude Design, regenerate, and replace `index.html` with the new export.

## Assets

`/assets/` contains the source PNG images (hero, book_transfer, read_together, sustainability, new_chapter) used during design. This directory is **not tracked in git** — the images are already embedded as base64 inside `index.html`.
